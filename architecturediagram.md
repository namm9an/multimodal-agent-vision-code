## Why This Architecture?

### Why Images Only (No PDF)?
PDF parsing requires OCR pipelines, multi-page handling, and table extraction â€” each is complex. Users can screenshot PDFs. Proper support in Phase 2.

### Why Python Only (No JS)?
Single runtime = one security policy, one set of dependencies. Python covers 95% of data analysis. JS adds attack surface with npm ecosystem.

### Why Critic Agent?
Low-effort, high-value. Static analysis catches bugs before sandbox runs. Reduces failed executions and improves user experience.

### Why No RAG?
Embedding pipelines, vector search, and retrieval tuning are each sub-projects. Session-only context is sufficient for MVP. Job metadata stored for history.

### Philosophy
> Start with the simplest thing that works. Add complexity only when metrics demand it.
